## Ground truth hierarchy

1. `SPEC.MD` — product scope and architecture decisions. If a task conflicts with SPEC, stop and flag it; do not silently deviate.
2. `docs/agents/*` — how to build.
3. `docs/processes/*` — how changes move (branching, CI, release).

## Quick orientation

- One deployable: `apps/web` (Next.js App Router, UI + Route Handlers).
- All real logic lives in `packages/{db,auth,ai,shared}`; Route Handlers are thin adapters.
- Bun = package manager & task runner. Node.js = production runtime. Never introduce Bun-runtime-only APIs into app code.
- Postgres via Drizzle; migrations are forward-only and generated by drizzle-kit.
